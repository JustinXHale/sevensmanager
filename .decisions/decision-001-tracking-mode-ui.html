Tracking Mode UI Pattern

Visual

Right now, the Full vs Simple mode toggle looks like tabs, but it's really a preference setting that changes how event logging works. Simple mode is 1-tap logging (tap "P" and it's logged), while Full mode opens zone pickers for detailed position tracking. Users need to understand what each mode does before choosing, and the UI should feel like a setting, not navigation.

Classic Toggle Switch

Simple
Full
A familiar iOS-style toggle switch with "Simple" on the left and "Full" on the right. When toggled, the knob slides and the active label is highlighted. This is the most compact option and works well for a binary choice. It clearly signals "this is a setting," not navigation, which solves your concern about the tab-like appearance.

Works well when

  • Users understand what Simple/Full mean
  • Space is limited on mobile
  • You want the lightest visual weight

Watch out for

  • Doesn't explain what each mode does
  • Users have to remember which is which

Segmented Control with Labels

Simple
Quick log
Full
With details
A segmented control (think iOS settings) where both options are always visible side-by-side. Each segment has a title ("Simple" / "Full") and a small subtitle ("Quick log" / "With details") that explains what it does. The active choice has a white pill behind it. This tells the story better than just "Simple" or "Full" alone, helping users understand the trade-off before they tap.

Works well when

  • Users need hints about what each mode does
  • You want both options visible at once
  • Mobile-friendly and thumb-reachable

Watch out for

  • Takes more horizontal space than a switch
  • Subtitle text must stay very short

Zone details
Two cards side-by-side, each with an icon, title, and short description. Tapping a card selects it (border turns green, background tints). The icons give a quick visual: lightning bolt for quick Simple mode, pin for detailed Full mode. This gives you more room to explain the difference, and feels like picking a preference rather than navigating between tabs.

Works well when

  • You want to be very explicit about the difference
  • Icons help communicate the concept
  • Users are choosing once and staying put

Watch out for

  • Takes up the most vertical space
  • Might feel too prominent for a small toggle

Compact Dropdown

Tracking mode
Full mode (with details)
A small labeled dropdown that says "Tracking mode" with the current selection showing ("Full mode (with details)" or "Simple mode (quick log)"). When tapped, it opens a menu with both options and their descriptions. This is the most compact option and keeps the setting out of the way. Good if users set it once and rarely change it, but less discoverable than the other options.

Works well when

  • Users set it once at the start of a match
  • You want minimal UI footprint
  • Screen space is at a premium

Watch out for

  • Hidden until clicked (less discoverable)
  • Requires an extra tap to see options
