# SevensManager

Rugby sevens squad and match analytics — **PWA**, local-first (IndexedDB). Product and build plans live in [`docs/PRODUCT_SPEC.md`](docs/PRODUCT_SPEC.md) and [`docs/IMPLEMENTATION_SPEC.md`](docs/IMPLEMENTATION_SPEC.md).

**Display name:** `SevensManager` by default. Override with `VITE_APP_NAME` in `.env` for builds (see `src/config/appMeta.ts`).

**IndexedDB:** the app database is named `sevensmanager`. If you previously ran an older clone under the old database name, local data may not carry over — clear site data or use a fresh browser profile.

## PWA / offline

- Service worker and manifest are generated by **vite-plugin-pwa** on **`npm run build`**.
- To exercise install + offline: `npm run build && npm run preview`, open the printed URL on Android Chrome → **Add to Home screen**.

**Phase 1** (current): **Matches** (create / list / delete) with metadata (title, optional **our team** + opponent, **kickoff**, **location**, competition). **Add game** (`/matches/new`) saves one row at a time (same fields as CSV). **Import schedule** (`/matches/import`) pastes **CSV** to load many rows. **Two halves**, **running clock** — persisted in IndexedDB (`schemaVersion` 2). Saves return to the **match list**; tap a row to open the clock. Rosters and events are **Phase 2+**.

### Schedule import (CSV)

Export **comma-separated** from a spreadsheet with a **header row**. Supported columns include `title`, `ourTeamName`, `opponentName`, `kickoffDate`, `location` (or `venue`, `ground`, …), `competition` (aliases like `home` / `away` work). See **Load example** on the import screen.
